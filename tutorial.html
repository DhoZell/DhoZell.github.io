01 — Persiapan

Persiapan & Bikin File Project

🖥️
Pakai Editor Kode (Wajib)

Jangan nulis kode di Notepad biasa. Kamu butuh code editor yang punya syntax highlighting dan autocomplete.

  • Visual Studio Code (VS Code) — Rekomendasi utama. Gratis, ringan, banyak extension. Download di code.visualstudio.com
  • Sublime Text — Alternatif, juga gratis untuk basic use
  • Notepad++ — Minimal tapi lumayan buat Windows
✅ Install VS Code, lalu install extension Live Server (klik ikon kotak di sidebar kiri VS Code, cari "Live Server"). Extension ini bikin kamu bisa preview website real-time tanpa refresh manual.

Setelah install editor, buat folder baru di komputer kamu. Di dalam folder itu, bikin 3 file berikut.

STRUKTUR FOLDER 
my-project/
├── index.html     <-- halaman utama
├── style.css      <-- tampilan visual
└── script.js      <-- logika & interaksi
ℹ️ File utama HARUS bernama index.html (huruf kecil semua). GitHub Pages otomatis buka file ini saat link dibuka.
📄
index.html — Struktur Halaman

HTML adalah kerangka website. Semua konten (teks, gambar, tombol) ditulis di sini.

HTML 
<!DOCTYPE html>
<html lang="id">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Website Pertamaku</title>
  <!-- Hubungkan CSS -->
  <link rel="stylesheet" href="style.css">
</head>
<body>

  <header>
    <h1>Halo, Dunia! 👋</h1>
    <p>Ini website pertama saya yang live di internet.</p>
  </header>

  <main>
    <button id="btn">Klik Saya</button>
    <p id="pesan"></p>
  </main>

  <!-- Hubungkan JS di akhir body -->
  <script src="script.js"></script>
</body>
</html>
  • Tag <head> berisi metadata, judul tab, dan link ke CSS
  • Tag <body> berisi semua konten yang tampil di layar
  • Script JS ditaruh di bawah sebelum </body> supaya HTML dimuat dulu
🎨
style.css — Tampilan Visual

CSS mengatur warna, ukuran, layout, dan animasi. Tanpa CSS, website kamu bakal polos kayak Word document.

CSS 
/* Reset bawaan browser */
* {
  margin: 0;
  padding: 0;
  box-sizing: border-box;
}

body {
  font-family: 'Segoe UI', sans-serif;
  background: #f5f5f5;
  color: #222;
  padding: 40px;
}

header {
  text-align: center;
  margin-bottom: 40px;
}

h1 {
  font-size: 48px;
  margin-bottom: 12px;
}

button {
  display: block;
  margin: 0 auto;
  padding: 12px 28px;
  background: #6366f1;
  color: white;
  border: none;
  border-radius: 8px;
  font-size: 16px;
  cursor: pointer;
  transition: background 0.2s;
}

/* Hover effect */
button:hover {
  background: #4f46e5;
}

#pesan {
  text-align: center;
  margin-top: 20px;
  font-size: 18px;
  color: #6366f1;
}
  • Selector (* artinya semua elemen, h1 artinya khusus elemen h1)
  • Property seperti color, background, font-size adalah yang diubah
  • Nilai ditulis setelah titik dua dan diakhiri titik koma
⚡
script.js — Interaktivitas

JavaScript bikin website bisa merespons aksi pengguna. Tanpa JS, tombol kamu cuma pajangan.

JAVASCRIPT 
// Ambil elemen dari HTML menggunakan ID-nya
const btn = document.getElementById('btn')
const pesan = document.getElementById('pesan')

// Dengerin event 'click' pada tombol
btn.addEventListener('click', () => {
  pesan.textContent = 'Kamu berhasil klik tombol! 🎉'
  btn.style.background = '#16a34a'
})

// Contoh lain: ubah teks setiap detik
let count = 0
setInterval(() => {
  count++
  console.log(`Halaman sudah dibuka ${count} detik`)
}, 1000)
  • const dan let dipakai untuk menyimpan nilai (variabel)
  • addEventListener menunggu aksi pengguna (klik, ketik, hover)
  • console.log berguna untuk debug, tidak tampil ke pengguna
⚠️ Selalu cek browser console (F12) kalau ada error. Error JS biasanya ada di tab "Console".
02 — Akun

Daftar & Login GitHub

1
Buka github.com
Klik tombol "Sign up" di pojok kanan atas. Kalau sudah punya akun, klik "Sign in" aja.
2
Isi data pendaftaran
Masukkan email, buat username, dan password. Username ini penting karena jadi bagian dari link website kamu nanti.
💡 Pilih username yang profesional dan mudah diingat. Format link nanti: username.github.io/nama-repo
3
Verifikasi email
Cek inbox kamu, klik link verifikasi dari GitHub. Kalau nggak ada, cek folder spam.
4
Masuk ke dashboard
Setelah verifikasi, kamu langsung diarahkan ke dashboard GitHub. Siap lanjut ke langkah berikutnya.
03 — Repository

Buat Repository Baru

Repository (repo) adalah tempat penyimpanan file project kamu di GitHub. Anggap aja seperti folder project tapi di cloud.

1
Klik tombol "+"
Ada di pojok kanan atas, di sebelah foto profil kamu. Pilih "New repository".
2
Isi nama repository
Pakai huruf kecil dan tanda minus untuk spasi. Contoh: my-website, portofolio-2025, tugas-pemweb.
⚠️ Hindari spasi dan karakter spesial di nama repo. Pakai tanda minus (-) sebagai pemisah kata.
3
Set visibility ke Public
Wajib Public kalau mau pakai GitHub Pages gratis. Repository Private butuh akun berbayar untuk fitur Pages.
4
Centang "Add a README file"
Ini opsional tapi disarankan. README adalah file deskripsi project yang tampil di halaman repo.
5
Klik "Create repository"
Repo kamu sudah jadi! Sekarang waktunya upload file project.
04 — Upload File

Upload File ke GitHub

Ada 2 cara. Pilih salah satu yang paling nyaman buat kamu.

🖱️
Cara A — Upload Langsung di Browser

Cocok buat yang belum install Git. Nggak perlu terminal sama sekali.

  • Buka halaman repository kamu di GitHub
  • Klik tombol "Add file", pilih "Upload files"
  • Drag & drop semua file project (index.html, style.css, script.js) ke area upload
  • Di bagian bawah, tulis pesan commit, contoh: "upload project pertama"
  • Klik "Commit changes"
✅ Cara ini paling simpel. Cocok banget buat yang baru pertama kali.
💻
Cara B — Pakai Git di Terminal

Cara ini lebih powerful dan jadi kebiasaan programmer profesional. Install Git dulu dari git-scm.com.

TERMINAL / CMD / POWERSHELL 
# Step 1: Konfigurasi identitas (lakukan sekali saja)
git config --global user.name "Nama Kamu"
git config --global user.email "email@kamu.com"

# Step 2: Masuk ke folder project
cd my-project

# Step 3: Inisialisasi Git di folder ini
git init

# Step 4: Tambahkan semua file ke staging
git add .

# Step 5: Commit dengan pesan yang jelas
git commit -m "upload project pertama"

# Step 6: Hubungkan ke repo GitHub
# (copy URL dari tombol "Code" di halaman repo)
git remote add origin https://github.com/username/my-website.git

# Step 7: Upload ke GitHub
git branch -M main
git push -u origin main
  • git add . artinya tambahkan SEMUA file yang berubah
  • git commit adalah "simpan snapshot" dari kode kamu
  • git push adalah "upload" snapshot itu ke GitHub
🚫 PENTING: Sejak Agustus 2021, GitHub sudah TIDAK menerima password biasa untuk git push via HTTPS. Kamu WAJIB pakai Personal Access Token (PAT) atau SSH key. Lihat instruksi di bawah.
🔑
Cara Buat Personal Access Token (PAT)

PAT adalah pengganti password untuk autentikasi Git via terminal. Wajib dibuat kalau pakai HTTPS.

1
Buka Settings GitHub
Login ke github.com. Klik foto profil di pojok kanan atas. Pilih "Settings".
2
Masuk ke Developer Settings
Scroll ke bawah sidebar kiri Settings sampai nemu "Developer settings". Klik. Lalu pilih "Personal access tokens" → "Tokens (classic)".
3
Generate new token
Klik "Generate new token" → pilih "Generate new token (classic)". Isi Note (nama token, bebas, misal: "laptop-kantor"). Set Expiration sesuai kebutuhan. Centang scope "repo" — ini wajib supaya bisa push code.
4
Copy token SEKARANG
Klik "Generate token". Token muncul SEKALI SAJA. Langsung copy dan simpan di tempat aman (Notepad, password manager). Kalau halaman ditutup, token tidak bisa dilihat lagi dan harus bikin ulang.
🚫 Token ini setara password. Jangan share ke siapapun. Jangan ikut di-commit ke repo. Kalau bocor, hapus token itu di GitHub dan buat yang baru.
5
Pakai token saat git push
Saat terminal minta Password, paste token itu (bukan password akun GitHub). Terminal tidak menampilkan karakter apapun saat paste — itu normal, langsung Enter saja.
TERMINAL — LOGIN DENGAN TOKEN 
# Saat diminta credentials:
Username: username_github_kamu
Password: ghp_xxxxxxxxxxxxxxxxxxxx  <-- paste token di sini

# Biar tidak diminta ulang setiap push, jalankan ini:
git config --global credential.helper store
# Setelah push pertama berhasil, credentials tersimpan otomatis
💡 Alternatif lebih aman: pakai SSH key. Buka terminal, ketik ssh-keygen -t ed25519 -C "email@kamu.com", lalu tambahkan public key ke GitHub di Settings → SSH and GPG keys. Dengan SSH, tidak perlu input credentials setiap push.
05 — Deploy

Aktifkan GitHub Pages

Ini langkah terakhir. Setelah ini website kamu bisa diakses siapa aja.

1
Buka tab "Settings"
Di halaman repository, ada tab Settings di baris menu atas (paling kanan). Klik itu.
2
Cari menu "Pages"
Di sidebar kiri Settings, scroll ke bawah sampai nemu "Pages" di bawah bagian "Code and automation".
3
Pilih source branch
Di bagian "Build and deployment", set Source ke "Deploy from a branch". Lalu pilih branch "main" dan folder "/ (root)". Klik Save.
4
Tunggu 1–3 menit
GitHub butuh waktu untuk build website kamu. Refresh halaman Settings setelah beberapa menit.
5
Link kamu siap! 🎉
Muncul kotak hijau di bagian atas Pages dengan link website kamu. Format: username.github.io/nama-repo
🚀 Website kamu sekarang online dan bisa diakses siapa aja di seluruh dunia. Gratis. Selamanya selama akun GitHub aktif.
06 — Update

Cara Update Website

Website udah live tapi mau ubah isi konten atau tampilannya? Gampang.

🖱️
Update via Browser
  • Buka file di GitHub (klik nama filenya)
  • Klik ikon pensil (Edit) di kanan atas area kode
  • Edit langsung di browser
  • Scroll ke bawah, klik "Commit changes"
  • Website otomatis update dalam 1–2 menit
💻
Update via Git
TERMINAL 
# Edit file kamu dulu, lalu jalankan:

# Tambahkan perubahan
git add .

# Commit dengan pesan deskriptif
git commit -m "update: ubah warna header jadi biru"

# Push ke GitHub
git push
💡 Biasain tulis pesan commit yang jelas. "update" doang itu nggak helpful. Tulis yang spesifik kayak "fix: tombol login error di mobile" atau "feat: tambah halaman kontak".
07 — Troubleshoot

Error Umum & Solusinya

Ini error yang paling sering muncul. Jangan panik, semua ada solusinya.

❌
error: src refspec main does not match any

Muncul saat kamu belum pernah commit sama sekali sebelum push.

SOLUSI 
# Pastikan sudah ada file lalu add dan commit dulu:
git add .
git commit -m "first commit"
git push -u origin main
❌
remote: Repository not found / 403 Forbidden

Bisa berarti URL repo salah, atau token/credentials kamu salah.

SOLUSI 
# Cek remote URL yang tersimpan:
git remote -v

# Kalau salah, hapus dan set ulang:
git remote remove origin
git remote add origin https://github.com/username/nama-repo.git

# Kalau masalah credentials, hapus yang tersimpan lalu push ulang:
git config --global --unset credential.helper
❌
GitHub Pages: website tidak muncul / 404

Ada beberapa penyebab umum.

  • File utama tidak bernama index.html (case sensitive, huruf kecil semua)
  • File ada di dalam subfolder, bukan di root repo. Pastiin index.html langsung di level pertama repo
  • GitHub Pages belum selesai build. Tunggu 2–5 menit lagi lalu refresh
  • Source branch di settings Pages salah. Pastiin pilih branch "main", bukan "master"
  • Repo masih Private. Ganti ke Public dulu di Settings > General > Danger Zone
❌
fatal: branch 'main' tidak ada, muncul 'master'

Default branch di Git lama adalah "master", sedangkan GitHub sekarang pakai "main". Bisa konflik.

SOLUSI — Rename master ke main 
# Ganti nama branch dari master ke main:
git branch -M main

# Lalu push:
git push -u origin main

# Atau set default branch baru untuk semua repo ke depan:
git config --global init.defaultBranch main
⚠️
CSS atau JS tidak load di GitHub Pages
  • Pastiin path file di HTML pakai relative path, bukan absolute. Contoh benar: href="style.css" bukan href="/style.css"
  • Nama file case sensitive di server Linux GitHub. "Style.css" dan "style.css" dianggap beda file
  • Cek di tab Network di DevTools (F12) untuk lihat file mana yang gagal dimuat
08 — Ringkasan

Teknologi yang Kamu Pakai

Teknologi Fungsi Wajib?
HTML Struktur halaman. Konten, heading, paragraf, tombol, gambar. Ya
CSS Tampilan visual. Warna, font, layout, animasi, responsif. Ya
JavaScript Interaktivitas. Klik tombol, fetch API, animasi dinamis. Opsional
Git Version control. Lacak perubahan kode, simpan histori. Sangat disarankan
GitHub Cloud storage untuk kode. Kolaborasi, backup, portfolio. Ya
GitHub Pages Hosting gratis. Website static langsung bisa diakses publik. Ya (tujuan akhir)
⚠️ GitHub Pages hanya untuk website static (HTML, CSS, JS). Kalau website kamu butuh backend (Node.js, PHP, Python) atau database, kamu perlu hosting berbayar seperti Railway, Render, atau VPS.
